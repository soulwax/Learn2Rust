Learn2Rust is a course-as-repository for developers returning to code — you already know C#, Java, or TypeScript. You'll learn Rust by building Focus Forge, a real project, task, and note workbench, one chapter at a time: from tiny CLI labs to a desktop GUI.

Get started in three commands

You need Rust (via rustup) and, ideally, VS Code with the recommended extensions. Then clone and run the first lab from the repository root.

1

Check your tools

Confirm the Rust toolchain is installed and on your PATH.

2

Clone the repo

Grab the course and move into it. Everything runs from the root.

3

Run the first lab

Build, test, and run Chapter 0 — your first working Rust program.

# 1 · check your tools
rustc --version
cargo --version

# 2 · clone and enter the course
git clone https://github.com/soulwax/Learn2Rust.git
cd Learn2Rust

# 3 · run the first lab (from the repo root)
cargo run -p ch00_setup
cargo run -p ch00_setup -- YourName
Expected output: Hello, Rust! (or Hello, YourName!), followed by Chapter 0 is alive. Cargo, VS Code, and you are talking.

Verify the workspace

These four commands are the heartbeat of the course. Run them anytime — they check, test, format, and lint the entire Cargo workspace.

cargo check                                              # fast type-check, whole workspace
cargo test                                               # run every test
cargo fmt --check                                        # verify formatting
cargo clippy --workspace --all-targets --all-features    # lint

In VS Code, the same commands live under Terminal → Run Task (cargo: check workspace, cargo: test workspace, and friends).

How the workspace is laid out

One Cargo workspace is your home base. Real application code lives in crates/; small chapter experiments live in labs/. Every folder owns exactly one kind of thing.

Learn2Rust/
├─ crates/
│  └─ focus_forge_core/   # the domain: projects, tasks, notes, JSON storage
├─ labs/
│  └─ ch00_setup/        # your first runnable Rust lab
├─ chapters/            # the teaching narrative
├─ assignments/         # tasks, acceptance criteria, verification
├─ docs/                # references, troubleshooting, decision records
├─ sample_data/         # committed curriculum data
└─ Cargo.toml           # the workspace manifest

🧩 focus_forge_core

The heart of the app: domain types, validation, a single error type, and JSON persistence. It knows nothing about the CLI or GUI — they depend on it, never the other way around.

🔬 labs/chXX

Tiny, focused packages — one Rust concept each. A safe sandbox to experiment, break things, and read the compiler's feedback. Run any lab with cargo run -p <name>.

Handy Cargo commands

Scope any command to one package with -p, or drill down to a single test by name.

cargo run -p ch00_setup -- Soulwax            # run a lab with an argument
cargo check  -p focus_forge_core               # check just one crate
cargo test   -p focus_forge_core               # test just one crate
cargo test   -p ch00_setup greeting_uses_provided_name   # one test by name
